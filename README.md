# Fast GUI LLM Orchestration (Project Name: NodeAgent / 가칭)
**"Teachable Machine for AI Agents"** - 비개발자도 직관적으로 나만의 AI 에이전트를 만들고 소유하는 GUI 환경.

---

## 💡 1. 핵심 비전 및 차별점 (Vision & Differentiation)

### 1-1. "Teachable Machine" 컨셉의 극강의 직관성
복잡한 프레임워크(LangChain 등)의 개발자 친화적인 용어 대신, `키보드 텍스트 입력`, `문자열 결합`, `결과 출력` 등 비개발자도 이해할 수 있는 직관적이고 일상적인 언어로 된 노드를 제공합니다. '블록 놀이'하듯 노드를 이어 붙이는 것만으로 누구나 강력한 LLM 워크플로우를 프로토타이핑할 수 있습니다.

### 1-2. 무코드 독립 파이썬 코드 추출 (Standalone Python Export) - ⭐️ 킬러 기능
기존의 다른 노드형 AI 플랫폼 시스템들은 자사 서버에 종속(Platform Lock-in)되는 한계가 있었습니다. 본 프로젝트의 가장 큰 차별점은 **"생성물의 완전한 소유와 독립 실행"**을 보장한다는 것입니다.
* 에디터에서 UI로 제작을 완료하면, **`.py` (파이썬 파일)** 하나로 에이전트 생성물을 즉각 다운로드할 수 있습니다.
* 복잡한 거대 시스템을 설치할 필요 없이, 어디서든 사용자 본인의 API 키만 입력하면 **다운받은 파이썬 파일 단독으로 완벽하게 똑같이 실행**됩니다.
* 코드를 전혀 모르는 비개발자라도 자신만의 맞춤형 시스템을 파이썬 스크립트 형태로 소유하게 됩니다.

### 1-3. 에이전트 직렬화 및 경량 구조 (Agent Serialization)
사용자가 작업한 노드 캔버스의 연결 상태(DAG)는 가벼운 `JSON`(.agent) 객체로 직렬화(Serialization)되어 저장/배포될 수 있습니다. 이를 통해 클라이언트 엔진과 백엔드 엔진 간 완벽한 분리를 지향합니다.

### 1-4. 다양한 언어로의 이식 및 재구성 (Cross-Language Reshapability)
본 프로젝트의 핵심 아키텍처는 특정 언어에 종속되지 않습니다. 정의된 JSON 스펙만 있다면 Python뿐만 아니라 Java, JavaScript, 심지어 C++나 C와 같은 저수준 언어에서도 동일한 로직의 실행기(Executor)를 구현하고 재구성(Reshape)할 수 있는 범용적인 설계를 지향합니다.

## 🎯 2. 프로젝트 개요 및 타겟

### 2.1. 프로젝트 소개
Scratch의 블록 코딩, 언리얼 엔진의 매터리얼 노드/블루프린트, 혹은 ComfyUI의 노드 시스템처럼 시각적 인터페이스(Node-based GUI)를 통해 다수의 LLM 노드, 조건문, API 도구들을 손쉽게 엮고 테스트할 수 있는 프론트엔드 환경 및 파싱/실행 백엔드 엔진입니다.

### 2.2. 타겟 유저
*   **비개발자 기획자 / 크리에이터:** 코드 한 줄 없이 자신만의 로직이 담긴 나만의 AI 프로그램(스크립트)을 만들고 싶은 유저.
*   **프롬프트 엔지니어:** 다양한 변수와 조건이 복잡하게 얽힌 프롬프트 체이닝을 시각적으로 빠르게 검증하고 싶은 전문가.

## 🛠 3. 핵심 기능 (Core Features)

### 3.1. 직관적 노드 에디터 환경
*   **입/출력 노드:** 키보드 텍스트 입력, 파일 읽기, 최종 결과 출력 창 등.
*   **LLM 및 처리 노드:** OpenAI 등 다양한 LLM 연결, 프롬프트 템플릿(변수 치환), 문자열 결합/추출, 조건문 분기(If-Else).
*   **연결 및 타입 매칭:** 출력 포트와 입력 포트의 타입이 일치할 때만 연결되는 직관적인 선 연결.

### 3.2. 실시간 구동 및 시각적 피드백 (Live Debugging)
*   Play를 누르면 노드 연결선을 따라 데이터의 흐름이 애니메이션으로 표현.
*   각 노드가 어떤 텍스트를 내놓았고 파싱했는지 중간 결괏값 즉각 확인 가능.

### 3.3. 엔진 파서 & 생성자 (Parser & Generator)
*   사용자의 시각적 흐름(JSON)을 파싱하여 순서대로 LLM API를 호출해 실행결과를 반환.
*   해당 JSON 논리구조를 기반으로 한 독자적인 실행 가능한 Python Script 자동 생성.

---

## 💻 4. 시스템 구조 및 추천 기술 스택

기존 서비스들의 한계를 넘기 위해 '결과물의 독립성'과 '경량성'을 최우선으로 둡니다.

*   **Frontend GUI (Visual Builder)**
    *   **프레임워크:** React (Vite 기반 설계)
    *   **핵심 라이브러리:** `React Flow` (노드 그래프 시스템의 UI/UX 구현 중심 뼈대)
    *   **스타일링:** TailwindCSS, Shadcn UI
*   **Backend & Compiler (Execution & Translation Engine)**
    *   **프레임워크:** FastAPI (Python)
    *   **엔진 역할:** 프론트에서 넘겨받은 `.agent` (JSON) 구성도를 분석하여, 1) 직접 결과를 실행해 주거나 2) 완전히 독립 실행되는 타겟 `.py` 파일을 합성 기능.
    *   **LLM 라이브러리:** 커스텀 Python 템플릿 혹은 LangChain Core 라이브러리에 대응.

## 🗺 5. 향후 개발 로드맵 (Roadmap)

### Phase 1: MVP 완성 (Visual Editor + Python Code Generator)
*   React Flow를 통한 기본 노드(Input, Prompt, LLM, Output) UI 구현 및 드래그 앤 드롭 기능 통합.
*   설계된 노드를 기반으로 JSON 객체 직렬화 기능.
*   가장 뼈대가 되는 "Node JSON → Python Code" 변환 템플릿 엔진 개발.

### Phase 2: 로직(Logic) 컴포넌트 추가 및 고도화
*   If-Else 조건문, 다중 분기, Loop(반복), 파일 저장(TXT/JSON) 노드 등을 추가.
*   UI 테마 디자인 점검 (게임 엔진 Blueprint 스타일의 몰입감 높은 다크 모드 GUI 탑재).

### Phase 3: 도메인 특화 템플릿 및 플랫폼화
*   "NPC 대화 생성기", "나만의 번역기" 등 바로 불러와 써볼 수 있는 기성 `.agent` 템플릿 샘플 제공.
*   외부 플러그인(웹 검색 도구 노드 등) 확장에 대비한 모듈 아키텍처 수립.
