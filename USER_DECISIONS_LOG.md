# User Decision Log: Fast GUI LLM Orchestration
이 문서는 프로젝트 진행 과정에서 **사용자(USER)가 내린 주요 기획 및 아키텍처 의사결정**을 날짜별로 기록하는 파일입니다. 향후 개발 세션에서도 이 문서를 열람하여 일관된 제품 방향성을 유지합니다.

## 프로젝트 계기 (Background)
> "2026년 봄학기, AI와 사회에 대한 산업디자인학과 강의를 듣던 도중, AI 기술을 사람들이 얼마나 올바르게 접하고, 사용하는 경험을 쌓아야지 그에 대한 이해도가 높아질 수 있는지 배우게 되었습니다. 저 또한 과거 딥러닝과 머신러닝이 인기이던 동안, Google의 Teachable Machine을 통해 다가가기 어렵기만하던 기술을 접하고, 사용하고, 그 잠재력을 실감할 수 있었습니다.
이제 AI를 통한 LLM과 에이전트의 시대가 온 지금, 비개발자들과 비전공자, 비전문가들 또한 LLM을 기반으로한 에이전트와 오케스트레이션의 쉽고 간단한 접근을 통해 빠른 프로토타입을 만들고 이용하는 경험이 필요하다고 느꼈고, 문득 떠오른 아이디어와 함께 GUI를 기반으로 노드 단위의 LLM 모델과 로직 노드, 문자열 노드, 포맷 노드 등을 조작하여 쉽고 빠르게 자신만의 필요에 맞는 LLM Agent를 만들게 되길 바랍니다."

## 2026-04-09
- 타겟 유저를 비개발자와 프롬프트 엔지니어로 잡고, 복잡한 코드 작성 없이 노드를 시각적으로 연결할 수 있는 GUI 환경을 기획하라.
- 단순히 기존 유사 서비스(Flowise, Langflow 등)를 따라하는 것을 넘어, 구글의 **'Teachable Machine'**처럼 누구나 쉽게 접근할 수 있는 방향으로 브랜딩 및 UI/UX를 설계하라.
- 완성된 노드 워크플로우를 서버에 종속시키지 말고, 사용자가 자신의 API 키만 입력하면 로컬에서 즉시 동작하는 **순수 파이썬 단독 파일(.py)로 추출(Export)하는 기능**을 이 프로젝트의 최우선 킬러 기능으로 구현하라.
- 프로젝트 전체 철학과 비전이 담긴 초기 문서를 `README.md`로 정리한 후 Github 저장소에 바로 Commit 및 Push하여 문서화를 확립하라.
- 프로젝트 진행 중 발생하는 사용자의 모든 기획적/기술적 의사결정을 프로젝트 내 별도의 기록용 문서(현재 이 문서)에 날짜별 명령문 형태로 영구 보존하라.
